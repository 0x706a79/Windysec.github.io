0x01 模数分解

n的分解

rsa最暴力最简单的方式就是分解n，将n分解成功，得到p、q的值，可求n的欧拉函数的值。

  • φ(n)=(p−1)(q−1)

e，d与n的欧拉函数满足如下关系：

  • ed=1 mod φ(n)

即在知道e，p，q的情况下，可以解出d：

  • c ≡ m^e mod n

在已知p，q的时候利用modinv函数求出了d，然后就可以利用pow函数求出明文：

m=pow(c,d,n)

大数分解网站http://factordb.com/index.php

windows平台的RSATool2v17

利用公约数

两次公钥的加密过程中使用的n1和n2具有相同的素因子，那么可以利用欧几里得算法直接将n1和n2分解。
通过欧几里得算法可以直接求出n1和n2的最大公约数p：

  • (n1,n2)=p

可以得出：

  • n1=pq1
  • n2=pq2

得到p，然后可以求得q，然后解得明文。

windows下可以使用yafu尝试分解n

0x03 加密指数

低加密指数攻击

在rsa中e也称为加密指数。由于e是可以随意选取的，选取小一点的e可以缩短加密时间，但是选取不当的话，就会造成安全问题。

e=3时便造成低加密指数攻击

例题：Extremely hard RSA

低加密指数广播攻击

如果选取的加密指数较低，并且使用了相同的加密指数给一个接受者的群发送相同的信息，那么可以进行广播攻击得到明文。

即，选取了相同的加密指数e（这里取e=3），对相同的明文m进行了加密并进行了消息的传递，那么有：

c1≡m^e mod n1

c2≡m^e mod n2

c3≡m^e mod n3

对上述等式运用中国剩余定理，在e=3时，可以得到：

cx≡m^3 mod n1n2n3

通过对cx进行三次开方可以求得明文。

识别：

这个识别起来比较简单，一般来说都是给了三组加密的参数和明密文，其中题目很明确地能告诉你这三组的明文都是一样的，并且e都取了一个较小的数字。

例题：

SCTF2016，CODE300

题目第二轮中通过流量包的方式给了广播攻击的参数。

例题：http://codezen.fr/2014/01/16/hackyou-2014-crypto-400-cryptonet

Coppersmith定理攻击

Coppersmith定理指出在一个e阶的mod n多项式f(x)中，如果有一个根小于n^1/e，就可以运用一个O(log n)的算法求出这些根。

这个定理可以应用于rsa算法。如果e = 3并且在明文当中只有三分之二的比特是已知的，这种算法可以求出明文中所有的比特。

ctf中的数据处理

rsa问题无非就是围绕c、m、e、d、n、p、q，但是一般题目不会直接给出这些数据。在做题之前，我们需要将给出的题目转换成可以做题的形式。

openssl

使用openssl生成密钥

  • openssl genrsa -out test.key 1024

将这个文件中的公钥提取出来

  • openssl rsa -in test.key -pubout -out test_pub.key

生成的公钥加密文件

  • openssl rsautl -encrypt -in flag -inkey test_pub.key -pubin -out hello.en

解密文件

  • openssl rsautl -decrypt -in hello.en -inkey test.key -out hello.de2

对于现成的.pub公钥文件,用openssl来进行分析
openssl asn1parse -in publickey.pub 分析格式
openssl asn1parse -in publickey.pub -i -strparse xxx 得到信息
或者已知是rsa的情况下,直接用
openssl rsa -in publickey.pub -pubin -modulus -text 来得到n
expoent也是同理的

pcap文件

针对此类文件可以使用wireshark follow一下。这种问题一般都是写了一个交互的crypto系统，所以可能产生多轮交互。

PPC模式

这种模式是上述pcap文件的交互版，会给一个端口进行一些crypto的交互，参数会在交互中给出。
